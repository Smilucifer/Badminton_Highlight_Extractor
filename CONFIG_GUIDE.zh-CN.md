# `my_config.json` 参数说明（中文）

本文档对应 `my_config.json` 的完整配置结构，参数默认值来自 `config.py` 当前实现，并补充一版适合 **Windows + CUDA + 平衡模式** 的调参建议。

## 推荐使用方式

- `my_config.json`：保留一份完整配置，作为你的可编辑主配置。
- 平衡模式建议：优先保持 `vision.use_half_precision=true`、`vision.num_workers=4`，在不明显牺牲稳定性的前提下兼顾速度。
- 如果后续发现 GPU 占用不高、CPU 仍有余量，可以再逐步提高 `vision.batch_size` 或 `export.max_workers`，不要一次改太多。

## `vision` 视觉分析

### `model_path`
- 默认值：`"yolov8n-pose.pt"`
- 作用：YOLO 模型文件路径，用于人物检测与运动分析。
- 调大/调小影响：不是数值参数，主要影响模型来源。
- 建议：保持默认，除非你换了模型文件名或路径。

### `batch_size`
- 默认值：`16`
- 作用：YOLO 一次推理多少帧。
- 调大：通常更快，但显存占用更高，显存不足时容易报错或降速。
- 调小：更稳，但吞吐下降。
- 平衡模式建议：`16` 起步；若显存 ≥ 8GB 且运行稳定，可试 `24`。

### `num_workers`
- 默认值：`4`
- 作用：把视频分块后并行做视觉分析的线程数。
- 调大：CPU 解码/调度更忙，可能更快，也可能因为争抢资源变慢。
- 调小：更稳、更省资源，但速度可能下降。
- 平衡模式建议：`4`，这是比较稳的 Windows 值。

### `sample_fps`
- 默认值：`5.0`
- 作用：每秒抽多少帧做运动分析。
- 调大：边界和运动变化更细，但耗时增加。
- 调小：更快，但容易漏掉短促变化。
- 平衡模式建议：`5.0`；如果你处理的是节奏特别快的高水平双打，可试 `6.0`。

### `confidence_threshold`
- 默认值：`0.5`
- 作用：检测框置信度阈值，低于这个值的人物检测会被丢弃。
- 调大：误检更少，但可能漏人。
- 调小：召回更高，但误检增多。
- 平衡模式建议：`0.5` 保持默认。

### `use_half_precision`
- 默认值：`true`
- 作用：是否启用半精度推理（FP16）。
- 调大/调小影响：开启通常更快、更省显存；关闭更保守。
- 平衡模式建议：`true`，前提是你的 CUDA 环境正常。

## `audio` 音频击球检测

### `sample_rate`
- 默认值：`null`
- 作用：`librosa` 读取音频时的采样率。`null` 表示使用其默认重采样策略。
- 调大：理论上保留更多细节，但多数情况下收益有限。
- 调小：更省资源，但可能损失短促击球细节。
- 平衡模式建议：保持 `null`。

### `hop_length`
- 默认值：`512`
- 作用：音频能量分析的步长。
- 调大：更快，但时间分辨率更粗。
- 调小：更细，但更慢。
- 平衡模式建议：`512`，比较稳。

### `peak_height`
- 默认值：`0.015`
- 作用：音频峰值检测阈值。
- 调大：更严格，少误报，也更容易漏掉轻击球。
- 调小：更灵敏，但噪声也更容易被当成击球。
- 平衡模式建议：`0.015`；如果场馆环境很吵，可试 `0.02`。

### `min_peak_distance_sec`
- 默认值：`0.2`
- 作用：两次峰值之间的最小时间间隔。
- 调大：会合并过密峰值，降低重复检测。
- 调小：更容易保留连续快拍，但也更易重复计数。
- 平衡模式建议：`0.2`。

## `fusion` 音视频融合切分

### `motion_percentile`
- 默认值：`50.0`
- 作用：用速度分布百分位数确定“剧烈运动阈值”。
- 调大：更严格，只有更明显的运动才算 intense。
- 调小：更宽松，容易保留更多回合，也更可能混入噪声。
- 平衡模式建议：`50.0`；想更稳可试 `55.0`。

### `hit_window_before`
- 默认值：`0.2`
- 作用：验证击球前，会往前看多少秒的运动数据。
- 调大：更容易吸收准备动作。
- 调小：更聚焦击球瞬间。
- 平衡模式建议：`0.2`。

### `hit_window_after`
- 默认值：`1.2`
- 作用：验证击球后，会往后看多少秒的运动数据。
- 调大：更容易确认这次音频峰值确实引发了持续对打。
- 调小：会更苛刻，也更可能错过持续时间短的有效回合。
- 平衡模式建议：`1.2`。

### `min_intense_ratio`
- 默认值：`0.05`
- 作用：击球附近窗口里，剧烈运动帧占比至少达到多少，才算有效击球。
- 调大：更严格，更干净。
- 调小：更宽松，更容易保留边缘案例。
- 平衡模式建议：`0.05`；如果误报偏多可试 `0.07`。

### `rally_gap_threshold`
- 默认值：`2.5`
- 作用：两次有效击球超过这个间隔，就切成两个回合。
- 调大：更容易把停顿算进同一回合。
- 调小：更容易拆开相邻回合。
- 平衡模式建议：`2.3` 到 `2.5`；通常 `2.4` 更平衡。

### `min_rally_duration`
- 默认值：`1.0`
- 作用：低于这个时长的回合会被丢弃。
- 调大：更偏向长回合高光。
- 调小：会保留更多短球。
- 平衡模式建议：`1.0`；如果你想多保留一些一拍到三拍的小回合，可试 `0.8`。

## `export` 导出

### `min_highlight_score`
- 默认值：`80`
- 作用：只有达到这个高光分数的回合才导出。
- 当前实现说明：现阶段分数核心仍和持续时间强相关。
- 调大：导出更少，但更偏长回合。
- 调小：导出更多，也更杂。
- 平衡模式建议：`80`；如果你想多出一些素材可试 `70`。

### `pad_front`
- 默认值：`1.5`
- 作用：导出时理论上的前置缓冲时间。
- 当前状态说明：v2.1 里主要边界由精修逻辑控制，这个值更多是历史兼容语义。
- 平衡模式建议：保持 `1.5`。

### `pad_back`
- 默认值：`2.0`
- 作用：导出时理论上的后置缓冲时间。
- 当前状态说明：同样主要受精修与 next serve protection 控制。
- 平衡模式建议：保持 `2.0`。

### `max_workers`
- 默认值：`5`
- 作用：并行导出片段的线程数。
- 调大：导出更快，但磁盘和 ffmpeg 并发压力更大。
- 调小：更稳、更安静。
- 平衡模式建议：`4` 到 `5`；普通 SSD 上 `4` 更稳。

## `boundary` 边界精修

### `start_search_window`
- 默认值：`2.0`
- 作用：起点向前搜索可信起始信号的窗口。
- 平衡模式建议：`2.0`。

### `end_search_window`
- 默认值：`2.0`
- 作用：终点向后搜索可信结束信号的窗口。
- 平衡模式建议：`2.0`。

### `max_front_adjustment`
- 默认值：`2.5`
- 作用：起点最多允许向前扩多远。
- 平衡模式建议：`2.5`。

### `max_back_adjustment`
- 默认值：`2.5`
- 作用：终点最多允许向后扩多远。
- 平衡模式建议：`2.5`。

### `max_start_trim`
- 默认值：`0.5`
- 作用：起点最多允许向后裁掉多少，避免过度截短准备动作。
- 平衡模式建议：`0.5`。

### `max_end_trim`
- 默认值：`0.5`
- 作用：终点最多允许向前裁掉多少，避免过度截短收拍。
- 平衡模式建议：`0.5`。

### `low_motion_percentile`
- 默认值：`25.0`
- 作用：用于估算低运动阈值，辅助识别进入/退出活跃状态。
- 调大：更容易把普通运动也视为低运动以上。
- 调小：更保守。
- 平衡模式建议：`25.0`。

### `pre_hit_guard`
- 默认值：`0.15`
- 作用：起点保护窗口，避免过度越过击球前关键瞬间。
- 平衡模式建议：`0.15`。

### `post_hit_guard`
- 默认值：`0.2`
- 作用：终点保护窗口，也用于防止吃进下一回合发球。
- 平衡模式建议：`0.2`；如果串球仍发生，可试 `0.25`。

### `fallback_pad_front`
- 默认值：`1.5`
- 作用：找不到可信信号时，起点默认往前补多少。
- 平衡模式建议：`1.5`。

### `fallback_pad_back`
- 默认值：`2.0`
- 作用：找不到可信信号时，终点默认往后补多少。
- 平衡模式建议：`2.0`。

### `state_window`
- 默认值：`0.35`
- 作用：比较状态变化时使用的局部时间窗。
- 调大：更平滑，但变迟钝。
- 调小：更敏感，但更抖。
- 平衡模式建议：`0.35`。

### `motion_rise_delta`
- 默认值：`15.0`
- 作用：判定起点附近是否出现明显运动上升。
- 调大：更严格。
- 调小：更灵敏。
- 平衡模式建议：`15.0`。

### `motion_fall_delta`
- 默认值：`15.0`
- 作用：判定终点附近是否出现明显运动下降。
- 调大：更严格。
- 调小：更灵敏。
- 平衡模式建议：`15.0`。

### `signal_match_min_score`
- 默认值：`2.0`
- 作用：边界候选点要达到的最小可信分数。
- 调大：只有更强证据才会认定为 signal-matched。
- 调小：更容易接受信号锚点。
- 平衡模式建议：`2.0`；如果你发现 fallback 太多且肉眼看边界其实还行，可试 `1.5`。

## 推荐调参顺序

如果你后面要继续调，不要全局乱拧，建议按这个顺序来：

1. 先看导出量对不对：调 `export.min_highlight_score`
2. 再看回合切分松不松：调 `fusion.rally_gap_threshold`、`fusion.min_intense_ratio`
3. 再看边界是不是总串球：调 `boundary.post_hit_guard`
4. 最后才碰性能项：`vision.batch_size`、`vision.num_workers`、`export.max_workers`

这样最不容易把问题搅成一锅粥。
